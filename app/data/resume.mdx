## 소개

주로 웹 애플리케이션의 프론트엔드를 개발함으로써 우리가 개발하는 서비스를 더 많은 사람들이 찾을 수 있게 하기 위해 필요한 작업을 진행했습니다.

사용자가 웹 애플리케이션을 이용할 때 발생하는 문제점을 해결하는 것을 중요하게 생각합니다. QA 및 디자인 담당 팀원으로부터 피드백을 반영하여 사용자에게 더 친화적인 프론트엔드를 개발하는 방법을 계속 생각해왔습니다.

투명한 커뮤니케이션을 지향합니다. 팀원으로부터 업무 시 고쳐야 할 점이 있으면 빠르게 반영하여 바람직한 개발 습관을 들입니다.

프로젝트를 시작할 때는 팀원들과 프로젝트 구조를 어떻게 형성해야 할지를 우선적으로 진행했습니다.

- 프로젝트 폴더 구조
- Github으로 프로젝트 진행상황을 관리할 때 브랜치 네이밍
- 서버 데이터 상태관리
- 클라이언트 내부 데이터 상태관리
- UI 테스트
- 라이브러리 선택 기준

예를 들어 라이브러리를 선택할 때는 주 다운로드 횟수가 몇 명 이상인지, 해당 라이브러리가 지속적으로 관리 및 개선되고 있는지, 공식문서와 예제가 자세하게 작성되어 있는지, 현재 존재하는 라이브러리들과 호환되는지를 빠짐없이 검토한 다음 라이브러리의 도입을 결정했습니다.

## 경력 사항

**Chromatic Protocol**

보유 중인 자산을 투자, 또는 유동성으로 예치할 수 있는 탈중앙화 암호화폐 선물 거래소

기술 스택

- 언어: TypeScript
- 라이브러리: React, React Router Dom, Vite, Redux Toolkit, SWR, GraphQL
- 프레임워크: Storybook, Tailwind CSS
- 기타: Patch Package, GraphQL Codegen, Github Action, Netlify Edge Functions, Vite Plugin PWA

작업 사항

- React, TypeScript 기반의 실시간 암호화폐 선물 거래소 UI 개발
- Optimistic UI를 적용하여 백엔드 또는 스마트 컨트랙트에 HTTP 요청을 전송하고 나서 응답 데이터 및 갱신된 데이터가 반환되기 전 예상되는 결과값을 화면에 우선 적용하여 반응성을 높일 수 있었음
- Storybook으로 개발 환경에서 UI 요소가 시나리오대로 동작하는지 테스트
- Chromatic으로 UI 컴포넌트를 웹으로 배포하여 디자인 팀원들이 각 UI 요소를 검토할 수 있었음
- React Suspense, SWR로 서버 및 컨트랙트로의 요청을 절약하고 데이터를 효율적으로 관리할 수 있었음
- 모바일, 태블릿 화면 사이즈 대응
- Patch Package로 팀에서 관리 중인 GraphQL 라이브러리의 이슈를 우선 조치하고 협업
- 브라우저에 저장된 CSS, JS 파일이 남아있을 때 프론트엔드가 새로 업데이트되면 간헐적으로 오류가 발생하는 이슈가 있었음
  - PWA 플러그인으로 프론트엔드가 업데이트되면 사용자에게 업데이트 알림
  - ErrorBoundary로 페이지에 에러가 발생하면 자동 새로고침하여 사용자의 혼란을 줄일 수 있었음
